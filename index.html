Hello this is a simple vanilla js pwa with a markdown blog

It only uses VanillaJS, Fetch API, Snarkdown(simple light md parser) and your browser.

What it uses?

It only uses pure, plain, simple vanilla JavaScript!, A bit of CSS! HTML just for... you know, works on browser. LOL

How it is made?

No pre/post/wibbly-wobbly processor for ANYTHING

It is just simple as:

  1. A JSON file describing post's name and file
  2. A JS to know whether to show the post's list or the post articles, based on anchor #id-of-section. As if it was a 'native pwa'.
  3. A JS to know when the #id-of-section is in use to:
    1. Valide the url after the hash to know if it renders the post's list or the post
    2. Get the list of post based on the json file
    3. Get the post content
    4. Parse and render the markdown content into html
  4. Fetch API for request both json file and the post article file when it's requested. In order to simplify its use, i'm using a fetch api wrapper, pure js also.
  5. A Simple, light, fast markdown parser. I've use Snarkdown
  6. Coffee

Oh wait, that is in fact how it was.

Looking up in the source files here you will see, i got these structure:

  • /assets
    /blog
    Here is the folder where i save all the .md files.
    blog.json
    Here is a simple json object with the post's array, each object has the properties: title (for showing) and name (the file name to request)
  • /scripts
    blog.js
    The one who takes care of the blog rendering, fetching, parsing and so on.
    content.js
    The one who takes care of what it shows on page (i'm current using because i made this example based on what i used on my website)
    Request.js
    The fatch api wrapper to simplify the code
    snarkdown.js
    The markdown parser
  • /style
  • index.html
  • main.js

How to use?

If you want to use it as i used or if you want to use it as initial idea to improve it as you need, you will have to:

Copy these whole source, really, all here, just copy, adjust the css as you need. Made the blog.html you indes.html.

You will have to do 3 important things exactly the same:

  1. Have the assets folder inside root folder
  2. Have the blog folder inside assets folder
  3. Have the blog.json file inside assets folder

Than you just import the content module and init it, as in main.js, it's really, really simple, has no secret or magic at all.
